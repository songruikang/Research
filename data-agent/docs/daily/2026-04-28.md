## 今日教训

### 1. 流程拆解比 prompt 优化更有效

WrenAI 的图表生成用 244 行 prompt 教 LLM 同时做选型+理解+生成，效果差。拆成四步后，Profiler 和 Selector 完全不需要 LLM，Generator 的 prompt 反而更短但更准——因为职责单一了。**先拆流程，再优化每步。**

### 2. 先 mock 后 LLM

用 `--mock` 模式先验证 profiler + selector 的选型是否合理，不依赖 LLM 就能发现大部分问题。LLM 调用不稳定（限流/超时），mock 测试能保证开发节奏不被阻塞。

### 3. 并行派发 subagent 效率高

10 个 task 通过 subagent-driven development 执行，独立 task 并行派发（Task 2+3 并行、Task 4+5+6 并行、Task 7+8+9+10 并行），总耗时约 15 分钟完成整个模块。
